# MoneyPrinterV2 — Video Engagement Upgrade

## What This Is

MoneyPrinterV2 is a Python CLI tool that auto-generates and uploads YouTube Shorts — from topic discovery through LLM scripting, image generation, TTS narration, subtitle rendering, and YouTube API upload. This milestone focuses on making the generated English-language clips more engaging: stronger hooks, motion on images, scene transitions, and more energetic TTS delivery.

## Core Value

Every generated clip must stop the scroll within the first 3 seconds — hook + motion + voice must work together to retain viewers.

## Context

- **Current pain**: Generated clips feel flat — static images, monotone TTS voice, weak opening line, hard cuts
- **Existing codebase**: Pillow is already installed (Ken Burns available in Python layer); Remotion handles final render in TypeScript/React; TTS is kittentts wrapping edge-tts SSML
- **Remotion constraint**: All visual effects (Ken Burns, transitions) must be implemented inside `remotion/src/` as React/Remotion components — Python only stages assets and passes props JSON
- **Hook injection point**: Script is generated by `YouTube.generate_script()` via Ollama; hook sentence must be pre-pended or made the first LLM output
- **TTS prosody**: edge-tts supports SSML `<prosody rate="..." pitch="...">` tags; kittentts wrapper may need updating to pass these through

## Constraints

- **Tech stack**: Remotion for all visual effects — no FFmpeg-based transitions, no MoviePy (fallback only)
- **TTS provider**: Stay on edge-tts (kittentts) — no new paid API keys required
- **LLM**: Ollama local — hook generation uses same provider, no external API
- **Asset pipeline**: All images staged to `remotion/public/assets/` before render; props JSON passed via file to `scripts/render.mjs`
- **Platform**: Windows 10 with bash shell; Node subprocess needs `shell=True` for npx
