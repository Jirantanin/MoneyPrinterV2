## Phase Details

### Phase 4: Module Scaffold
**Goal**: The Podcast pipeline is accessible from the main menu and all config/class wiring is in place for downstream phases to build on
**Depends on**: Nothing (first phase of v2.0)
**Requirements**: MOD-01, MOD-02, MOD-03
**Success Criteria** (what must be TRUE):
  1. User sees a "Podcast" option in the main menu and can select it without errors
  2. `src/classes/Podcast.py` exists as a standalone class that imports shared infrastructure (llm_provider, Gemini image gen, edge-tts, YouTube API) without modifying YouTube.py
  3. `config.json` accepts `podcast_narrator` object (name, persona, tts_voice, tts_rate) and `podcast_style_prompt` string without schema errors
  4. Running the Podcast menu option with an incomplete pipeline gracefully reports "not yet implemented" rather than crashing
**Plans**: 2 plans
Plans:
- [x] 04-01-PLAN.md — Create Podcast.py stub class with four-step interface
- [x] 04-02-PLAN.md — Wire Podcast into main menu, constants, and config

### Phase 5: Script Generation
**Goal**: User can generate a coherent 14-scene podcast script for any topic, with consistent narrator voice across all acts
**Depends on**: Phase 4
**Requirements**: SCRP-01, SCRP-02, SCRP-03, SCRP-04
**Success Criteria** (what must be TRUE):
  1. For a given topic, the pipeline produces exactly 14 scenes structured as intro(1) + act1(4) + act2(4) + act3(4) + outro(1)
  2. Each scene object contains both a `narration` field (spoken text) and an `image_prompt` field (visual description)
  3. Each act is generated with a running summary of prior acts injected into the prompt, producing narratively connected acts even for long topics
  4. The narrator persona (name, tone, style) from `podcast_narrator` config is visibly reflected in the script voice and is consistent across all 14 scenes
**Plans**: 1 plan
Plans:
- [ ] 05-01-PLAN.md — Implement generate_script() with 3-call LLM loop, summary injection, and JSON persistence

### Phase 6: Scene Assets
**Goal**: Every scene in the script has a generated image and synthesized audio file ready for the render pipeline
**Depends on**: Phase 5
**Requirements**: IMG-01, IMG-02, IMG-03, TTS-01, TTS-02, TTS-03
**Success Criteria** (what must be TRUE):
  1. After asset generation, `.mp/podcast_<id>/` contains exactly 14 image files and 14 audio files (one per scene)
  2. Every generated image visually reflects the dark comic / noir style lock prefix from `podcast_style_prompt`, not a default Gemini style
  3. Audio files use the voice configured in `podcast_narrator.tts_voice` (default: en-GB-RyanNeural) at the rate configured in `podcast_narrator.tts_rate` (default: -20%)
  4. Asset generation is resumable — if some scenes already have files, those scenes are skipped rather than re-generated
**Plans**: 3 plans
Plans:
- [x] 06-01-PLAN.md — Create src/image_provider.py (standalone Gemini image gen module)
- [x] 06-02-PLAN.md — Extend Tts.synthesize() with optional voice/rate overrides
- [x] 06-03-PLAN.md — Implement Podcast.generate_assets() and refactor YouTube.py to use image_provider

### Phase 7: FFmpeg Render Pipeline
**Goal**: The 14 scene clips are rendered and concatenated into a single MP4 that falls within the 8-10 minute target
**Depends on**: Phase 6
**Requirements**: REND-01, REND-02, REND-03
**Success Criteria** (what must be TRUE):
  1. Each of the 14 scenes is rendered as an individual MP4 clip combining its image and audio with an FFmpeg `zoompan` filter (Ken Burns slow zoom-in or zoom-out)
  2. All 14 clips are concatenated via FFmpeg into a single final MP4 at `.mp/podcast_<id>/final.mp4`
  3. Final video duration is between 8 and 10 minutes given 14 scenes of typical narration length
**Plans**: 1 plan
Plans:
- [ ] 07-01-PLAN.md — Implement Podcast.render() — FFmpeg zoompan per-scene clips + concat to final.mp4
- [ ] 07-01-PLAN.md — Implement Podcast.render() with FFmpeg zoompan per-scene clips and concat into final.mp4

### Phase 8: Thumbnail & Upload
**Goal**: The completed episode — thumbnail and final video — is uploaded to YouTube with appropriate long-form metadata
**Depends on**: Phase 7
**Requirements**: THUMB-01, THUMB-02, UPLD-01, UPLD-02
**Success Criteria** (what must be TRUE):
  1. A thumbnail image is generated via Gemini API in dark comic style derived from the episode topic, producing a distinct cover image per episode
  2. Thumbnail is successfully set on the YouTube video via the YouTube API v3 thumbnails endpoint (visible in YouTube Studio)
  3. Final MP4 is uploaded to YouTube using existing API v3 infrastructure and appears in the channel as an unlisted or public video
  4. Uploaded video metadata includes an LLM-generated title, description, and tags with `made_for_kids: false` and a long-form appropriate category ID
**Plans**: 2 plans
Plans:
- [x] 08-01-PLAN.md — Generate metadata (LLM title/description/tags) and thumbnail (Gemini dark comic image)
- [x] 08-02-PLAN.md — Implement YouTube API v3 upload with thumbnail set
