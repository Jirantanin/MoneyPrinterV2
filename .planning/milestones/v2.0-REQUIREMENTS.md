# Requirements: MoneyPrinterV2 — Video Podcast Module

**Defined:** 2026-04-01
**Milestone:** v2.0
**Core Value:** Every generated clip must stop the scroll within the first 3 seconds — hook + motion + voice must work together to retain viewers.

## v2.0 Requirements

### Module Structure

- [x] **MOD-01**: User can invoke the Podcast pipeline from the main menu in `src/main.py`
- [x] **MOD-02**: Podcast logic lives in `src/classes/Podcast.py` as a standalone class, sharing `llm_provider`, Gemini image gen, edge-tts, and YouTube API upload from existing infrastructure
- [x] **MOD-03**: `config.json` supports `podcast_narrator` object (name, persona, tts_voice, tts_rate) and `podcast_style_prompt` string key

### Script Generation

- [x] **SCRP-01**: User can generate a podcast script for a given topic, structured as exactly 14 scenes: intro(1) → act1(4) → act2(4) → act3(4) → outro(1)
- [x] **SCRP-02**: Script is generated act-by-act — each act prompt includes a running summary of all previous acts to maintain narrative coherence without exceeding Ollama context limits
- [x] **SCRP-03**: Each scene contains two fields: `narration` (spoken text) and `image_prompt` (visual description for Gemini)
- [x] **SCRP-04**: Narrator persona (name, tone, style) from `podcast_narrator` config is injected into every LLM prompt to maintain consistent voice across episodes

### Image Generation

- [x] **IMG-01**: Each scene generates one image via Gemini API using the scene's `image_prompt` field
- [x] **IMG-02**: Every image prompt includes a fixed dark comic / noir style lock prefix from `podcast_style_prompt` config, applied before the scene-specific prompt
- [x] **IMG-03**: Generated scene images are staged to a working directory (`.mp/podcast_<id>/`) before render

### TTS & Audio

- [x] **TTS-01**: Each scene's `narration` text is synthesized via edge-tts using the voice from `podcast_narrator.tts_voice` (default: `en-GB-RyanNeural`)
- [x] **TTS-02**: TTS speaking rate is configurable via `podcast_narrator.tts_rate` (default: `-20%` for slow, deep pacing)
- [x] **TTS-03**: Per-scene audio files (WAV/MP3) are saved to the working directory alongside scene images

### FFmpeg Render Pipeline

- [ ] **REND-01**: Each scene is rendered as a video clip by combining its image and audio via FFmpeg, with a `zoompan` filter applied for Ken Burns effect (slow zoom-in or zoom-out)
- [ ] **REND-02**: All 14 scene clips are concatenated into a single final MP4 using FFmpeg concat
- [ ] **REND-03**: Final video duration falls within the 8–10 minute range given the fixed 14-scene structure

### Thumbnail

- [x] **THUMB-01**: A thumbnail image is generated via Gemini API using a dark comic style prompt derived from the episode topic and title
- [x] **THUMB-02**: Thumbnail is uploaded to YouTube via the existing YouTube API v3 thumbnails endpoint as part of the upload step

### YouTube Upload

- [x] **UPLD-01**: Final MP4 is uploaded to YouTube using existing `YouTubeUploader` / API v3 infrastructure
- [x] **UPLD-02**: Upload metadata includes LLM-generated title, description, and tags appropriate for long-form content, plus category ID and `made_for_kids: false` flag

## Future Requirements (v2.1+)

### Series Management

- **SERIES-01**: User can define multiple named podcast series, each with its own narrator persona and style prompt
- **SERIES-02**: Episodes are grouped by series in the cache JSON

### Scheduling

- **SCHED-01**: Podcast episodes can be scheduled for auto-generation and upload via the existing cron pipeline

### Subtitles

- **SUB-01**: SRT subtitle file is generated from TTS audio and burned into the final video via FFmpeg

## Out of Scope

| Feature | Reason |
|---------|--------|
| Remotion for podcast render | Remotion retained for Shorts only; FFmpeg chosen for long-form performance |
| ElevenLabs / paid TTS | Stay on edge-tts (no new API keys required) |
| Dynamic/animated subtitles | Deferred — motion already handled by Ken Burns zoompan |
| Thai-language podcast clips | Current milestone targets English narrator voice |
| Short-form output from Podcast pipeline | Shorts and Podcast are separate pipelines |
| Per-scene transition effects (slide/push) | FFmpeg concat with Ken Burns is sufficient; transitions add complexity |
