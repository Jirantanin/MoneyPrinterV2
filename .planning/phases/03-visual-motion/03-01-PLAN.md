**Step 3 — Create KenBurnsFrame sub-component.** Add this new component BEFORE ImageSlideshow. This sub-component is necessary so that `useCurrentFrame()` returns the frame relative to the TransitionSeries.Sequence context (not the global composition frame):
```typescript
const KenBurnsFrame: React.FC<{
  imgPath: string;
  variant: KenBurnsVariant;
  durationInFrames: number;
}> = ({ imgPath, variant, durationInFrames }) => {
  const frame = useCurrentFrame();
  const progress = frame / Math.max(durationInFrames, 1);

  let transform = '';
  switch (variant) {
    case 'zoom-in':
      transform = `scale(${interpolate(progress, [0, 1], [1.0, 1.05], { extrapolateRight: 'clamp' })})`;
      break;
    case 'zoom-out':
      transform = `scale(${interpolate(progress, [0, 1], [1.05, 1.0], { extrapolateRight: 'clamp' })})`;
      break;
    case 'pan-left':
      transform = `scale(1.05) translateX(${interpolate(progress, [0, 1], [0, -3], { extrapolateRight: 'clamp' })}%)`;
      break;
    case 'pan-right':
      transform = `scale(1.05) translateX(${interpolate(progress, [0, 1], [0, 3], { extrapolateRight: 'clamp' })}%)`;
      break;
  }

  return (
    <AbsoluteFill>
      <img
        src={staticFile(imgPath)}
        style={{
          width: '100%',
          height: '100%',
          objectFit: 'cover',
          transform,
          transformOrigin: 'center center',
        }}
      />
    </AbsoluteFill>
  );
};
```

**Step 4 — Rewrite ImageSlideshow.** Replace the entire ImageSlideshow component body with:
```typescript
const ImageSlideshow: React.FC<Pick<VideoProps, 'imagePaths' | 'durationInSeconds'>> = ({
  imagePaths,
}) => {
  const { durationInFrames: totalFrames } = useVideoConfig();

  if (imagePaths.length === 0) {
    return <AbsoluteFill style={{ backgroundColor: '#111' }} />;
  }

  const n = imagePaths.length;
  // Guard: ensure transition frames don't exceed half of per-image raw duration
  const perImageRaw = totalFrames / n;
  const tf = n > 1 ? Math.min(TRANSITION_FRAMES, Math.floor(perImageRaw / 2)) : 0;

  // Duration compensation: TransitionSeries shortens total by (n-1)*tf
  // so pad each image's duration to absorb the overlap
  const perImage = n > 1
    ? Math.round((totalFrames + (n - 1) * tf) / n)
    : totalFrames;

  return (
    <TransitionSeries>
      {imagePaths.map((imgPath, i) => (
        <React.Fragment key={i}>
          <TransitionSeries.Sequence durationInFrames={perImage}>
            <KenBurnsFrame
              imgPath={imgPath}
              variant={KB_CYCLE[i % KB_CYCLE.length]}
              durationInFrames={perImage}
            />
          </TransitionSeries.Sequence>
          {i < n - 1 && (
            <TransitionSeries.Transition
              presentation={slide({ direction: SLIDE_DIRS[i % SLIDE_DIRS.length] })}
              timing={linearTiming({ durationInFrames: tf })}
            />
          )}
        </React.Fragment>
      ))}
    </TransitionSeries>
  );
};
```

**Key details:**
- `perImage` uses the compensated formula from research: `Math.round((totalFrames + (n-1) * tf) / n)` so `n * perImage - (n-1) * tf ≈ totalFrames` (audio/video duration match).
- `tf` guard: `Math.min(TRANSITION_FRAMES, Math.floor(perImageRaw / 2))` prevents Remotion runtime error when transition exceeds adjacent sequence duration (edge case: very short clips).
- Ken Burns variant cycles via `KB_CYCLE[i % 4]` so consecutive images always differ (VIS-02).
- Slide direction cycles via `SLIDE_DIRS[i % 4]` for visual variety.
- `useCurrentFrame()` inside `KenBurnsFrame` returns sequence-local frame (Remotion context-aware), not global frame. This is why KenBurnsFrame must be a separate component.
- `objectFit: 'cover'` on the img prevents edge exposure during scale transforms (Pitfall 3 from research).
- Remove the `Sequence` import from the "remotion" import line since it is no longer used anywhere in the file after this rewrite. Keep all other imports.

**What NOT to change:** Lines 83-255 (GradientOverlay, CategoryBadge, SubtitleLayer, BreakingNewsTicker, VideoShort root component) remain exactly as they are. The VideoShort root component already passes `imagePaths` and `durationInSeconds` to ImageSlideshow — no wiring changes needed.
