Create `src/shorts_server.py` following the exact same architecture as `podcast_server.py`. Key adaptations:

**Module-level state:** `shorts[short_id]` dict with fields: youtube (YouTube instance), niche, topic, language, account, mode (auto/step), status, current_step, step_states, metadata, error, run_dir, video_path, cancelled.

**Pipeline steps (9 steps):**
0. Generate Topic
1. Generate Script
2. Generate Hook
3. Generate Metadata
4. Generate Image Prompts
5. Generate Images
6. TTS (Text-to-Speech)
7. Generate Subtitles
8. Combine Video (Remotion)

**Routes (mirror podcast_server.py pattern):**
- `GET /` — serve shorts_ui.html
- `GET /api/accounts` — return `get_accounts("youtube")` from cache.py (filter to id, nickname, niche, language only — no firefox_profile)
- `POST /api/generate` — accept `{account_id, niche, topic, language, mode}`. Look up account from cache by account_id. If topic is empty, it will be generated in step 0. Create timestamped run_dir in `.mp/` (like run_youtube.py does). Init YouTube instance. Start `_run_pipeline` in background thread. Return `{short_id}`.
- `GET /api/stream/{short_id}` — SSE event stream (copy podcast_server.py pattern exactly: cursor-based, 0.5s poll, keep-alive every 15s)
- `GET /api/episode/{short_id}` — full state (status, steps, metadata, video_url, etc.)
- `POST /api/approve/{short_id}` — step-by-step approval
- `POST /api/cancel/{short_id}` — cancel pipeline
- `POST /api/upload/{short_id}` — call `youtube.upload_video()` via asyncio.to_thread. Return `{video_url, success}`.
- `GET /api/shorts` — list recent shorts from `.mp/` dirs (scan for dirs containing `.mp4` files, return summaries sorted by mtime desc, limit 20)
- `GET /static/{short_id}/{filename}` — serve files from run_dir (images, video, audio). Security: basename only, no path traversal.

**_run_pipeline(short_id) — background thread:**

Important: The YouTube class methods set instance attributes as side effects (self.subject, self.script, self.metadata, self.images, self.tts_path, self.srt_path, self.video_path). The pipeline must call them in sequence. Use `_StdoutCapture` pattern from podcast_server.py to capture print output as SSE log events.

```python
def _run_pipeline(short_id):
    ep = shorts[short_id]
    # ... StdoutCapture setup ...

    # Need Pillow ANTIALIAS shim (same as run_youtube.py)
    from PIL import Image as _PILShim
    if not hasattr(_PILShim, "ANTIALIAS"):
        _PILShim.ANTIALIAS = _PILShim.LANCZOS

    # Mock selenium (not needed)
    from unittest.mock import MagicMock
    sys.modules.setdefault("selenium_firefox", MagicMock())

    account = ep["account"]
    youtube = YouTube(
        account["id"], account["nickname"],
        ep["niche"], ep["language"],
        run_dir=ep["run_dir"],
    )
    ep["youtube"] = youtube

    # Step 0: Topic (if user provided topic, use it directly)
    if ep.get("topic"):
        youtube.subject = ep["topic"]
    else:
        _run_step(short_id, 0, youtube.generate_topic)

    # Step 1: Script
    _run_step(short_id, 1, youtube.generate_script)
    # Step 2: Hook — prepend to script
    def _gen_hook():
        hook = youtube.generate_hook()
        if hook and hook[-1] not in ".?!":
            hook += "."
        youtube.script = hook + " " + youtube.script
    _run_step(short_id, 2, _gen_hook)
    # Step 3: Metadata
    _run_step(short_id, 3, youtube.generate_metadata)
    # ... etc for steps 4-8, with step_mode approval checks between each
    # Step 5 (images): loop over youtube.image_prompts, call generate_image for each
    # Step 6 (TTS): instantiate TTS(), call synthesize
    # Step 7 (subtitles): call generate_subtitles
    # Step 8 (combine): call combine_remotion() — set youtube.video_path
```

**short_id format:** `short_{timestamp}` where timestamp = `YYYYMMDD_HHMMSS` (matches run_dir naming).

**launch_shorts_server():** Same as `launch_podcast_server()` but port 8898.

**Important details:**
